Vie privée & données

Neurow a été conçu pour éviter l’architecture “SaaS classique” (serveur intermédiaire). Cela dit, vos données sont traitées par le fournisseur que vous choisissez (OpenAI ou Google Gemini). Cette page clarifie précisément ce qui se passe.

1) Flux réseau : pas de serveur Neurow

D’après le code, Neurow effectue des requêtes HTTPS directement vers :

  • OpenAI : endpoint https://api.openai.com/v1/responses
  • Gemini : endpoint https://generativelanguage.googleapis.com/...

Il n’y a pas de relay serveur Neurow.

2) Où sont stockées les clés API ?

Les clés sont stockées localement via :

  • Stockage local Office si disponible ;
  • sinon stockage local du navigateur ;
  • et un fallback mémoire en dernier recours.
Recommandation. Évitez de configurer une clé sur une session partagée (poste commun) sans protection OS adéquate.

3) Logs et cache : ce qui est conservé

Logs (diagnostics)

Les logs enregistrent surtout des métadonnées utiles au diagnostic : fonction appelée, fournisseur, modèle, durée, tokens, code d’erreur, etc. Ils sont conservés localement, avec rétention courte.

Cache

Le cache local conserve des réponses pour éviter les relances et économiser des tokens. Cela améliore beaucoup l’expérience Excel, mais cela signifie aussi :

  • si vous traitez des données sensibles, les réponses peuvent contenir des informations sensibles ;
  • vous pouvez vider le cache à tout moment depuis le panneau.

4) Côté fournisseur (OpenAI / Google)

Neurow ne contrôle pas la politique de rétention, d’abuse monitoring, ou de stockage du fournisseur. Pour OpenAI, Neurow envoie explicitement store: false dans les requêtes Responses API, pour fonctionner de manière stateless.

Références (à lire selon votre contexte de conformité) :

https://platform.openai.com/docs/guides/your-data
https://platform.openai.com/docs/api-reference/responses
https://ai.google.dev/gemini-api/docs/api-key

5) Bonnes pratiques “Excel + IA”

  • Ne transmettez pas de secrets, identifiants, données médicales ou données strictement confidentielles si ce n’est pas autorisé.
  • Préférez des contextes “minimaux” (juste les colonnes utiles) plutôt qu’une plage brute énorme.
  • Videz cache/logs en fin de session si nécessaire.
Besoin entreprise. Si vous avez un besoin de conformité (rétention zéro, contrôle avancé, etc.), consultez les options et programmes du fournisseur choisi (OpenAI/Google) ou utilisez une offre adaptée (ex : politiques spécifiques côté organisation).
