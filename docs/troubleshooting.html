Dépannage

Cette page recense les problèmes les plus fréquents et leurs correctifs, dans le contexte spécifique “Excel + fonctions IA”.

Checklist rapide

  1. Dans Excel : =AI.KEYSTATUS()
  2. Test : =AI.TEST()
  3. Dans le panneau : vérifiez fournisseur, modèle, clé, max tokens.

Erreur : “Clé API manquante”

Symptôme : la cellule renvoie une erreur, et/ou le panneau indique “key missing”.

Correctif : allez dans le panneau (Gemini/OpenAI), collez la clé, cliquez Enregistrer, puis Tester.

Aide : Obtenir une clé API.

Erreur : “API_ERROR”, quotas, ou crédit

  • Vérifiez votre facturation et vos limites côté fournisseur.
  • Essayez un modèle plus léger (mini/flash) et un max tokens plus bas.
  • Si vous lancez beaucoup de cellules en recalcul, réduisez la surface du calcul ou utilisez davantage le cache.

Erreur : modèle introuvable / non autorisé

  • Le modèle choisi dans le panneau peut ne pas être accessible sur votre compte.
  • Essayez un modèle standard, ou vérifiez que votre clé a le bon périmètre.

Erreur : effort de raisonnement refusé

Symptôme : erreurs OpenAI après modification du slider “effort”.

Correctif : revenez sur medium. La doc mentionne principalement low/medium/high.

Voir : Choisir un modèle.

Timeouts (réponses lentes)

  • Réduisez Max output tokens.
  • Réduisez le contexte (plage plus petite).
  • Utilisez un modèle plus rapide.
  • Évitez de recalculer en boucle (valeurs volatiles, etc.).

AI.WEB : pas de sources / web search non supporté

La recherche web s’appuie sur les outils natifs du fournisseur et du modèle. Si le modèle ne supporte pas l’outil, AI.Native peut revenir à une réponse sans grounding, ou renvoyer une justification.

  • Essayez un modèle explicitement compatible “web search”.
  • Vérifiez votre réseau (certaines entreprises filtrent les outils web).

Sorties “spill” qui débordent / #SPILL!

  • Libérez l’espace autour de la cellule (le spill a besoin de cellules vides).
  • Évitez de placer le spill dans un tableau structuré qui bloque la zone.
  • Si un spill renvoie trop de lignes, réduisez l’entrée (ou la consigne) et relancez.

Quand tout semble correct, mais la réponse est vide

  • Regardez le panneau → Logs → détails (code/message).
  • Si votre instruction exige du JSON strict, un modèle peut renvoyer un format non conforme (rare, mais possible). Dans ce cas, relancez ou simplifiez l’instruction.
Astuce. Si vous suspectez un cache “mauvais”, videz le cache depuis le panneau puis recalculer la cellule.
