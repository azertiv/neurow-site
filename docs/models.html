Choisir un modèle (OpenAI / Gemini)

Neurow vous laisse choisir un modèle dans le panneau pour chaque fournisseur. L’objectif n’est pas “le meilleur modèle” en absolu, mais le bon compromis qualité / vitesse / coût pour votre feuille Excel.

Règle simple

  • Flash / Mini / Nano : rapide et économique (traduction, classification, nettoyage, extraction simple, résumés).
  • Pro / (modèle plus puissant) : meilleur sur les tâches ouvertes (raisonnement long, synthèses complexes, génération de tables riches, formules avancées).

OpenAI : effort de raisonnement

Sur OpenAI, certains modèles acceptent un paramètre reasoning.effort (slider dans Neurow). Il guide le modèle sur la quantité de “raisonnement” interne avant de produire la réponse.

  • low : plus rapide, moins de tokens, parfois moins rigoureux.
  • medium : équilibré (souvent le défaut).
  • high : plus approfondi, mais plus lent et plus coûteux.
Concrètement. Les modèles de raisonnement introduisent des “reasoning tokens” en plus des tokens d’entrée/sortie. Quand vous montez l’effort, vous payez souvent davantage pour une réponse plus robuste.
Valeurs supportées. La documentation OpenAI présente principalement low/medium/high. Si vous sélectionnez une valeur non supportée par le modèle, l’API peut refuser la requête. Si vous avez un doute : revenez sur medium.

Documentation officielle OpenAI :

https://platform.openai.com/docs/guides/reasoning

Gemini : “Flash” vs “Pro”

Côté Gemini, la logique est similaire : les variantes “Flash” privilégient latence et coût ; les variantes “Pro” privilégient robustesse et qualité quand le prompt est plus exigeant ou riche en contexte.

Référence Google (génération de contenu, outils, mode JSON, etc.) :

https://ai.google.dev/api/generate-content

Comment décider concrètement ?

1) À partir de la fonction

  • AI.TRANSLATE / AI.CLASSIFY / AI.CLEAN / AI.CONSISTENT : modèles rapides.
  • AI.FORMULA / AI.TABLE / AI.WEB : plus sensibles à la qualité ; testez un modèle plus puissant si les sorties sont insuffisantes.

2) À partir de la “surface” Excel

  • Cellule unique “décision” (ex : génération de formule finale) : qualité d’abord.
  • Grandes plages (ex : traduction de centaines de lignes) : coût/latence d’abord, et cache.

3) À partir de votre budget

Neurow affiche une estimation de coûts dans le panneau. Utilisez-la comme tendance, pas comme comptabilité exacte : les prix et règles changent côté fournisseurs.

Recommandations par cas d’usage

Cas Modèle Effort
Traduction / nettoyage / tags Flash/Mini low / medium
Classification stricte (labels) Flash/Mini low / medium
Résumé d’un document Mini/Pro medium
Formule Excel avancée (LET/LAMBDA) Plus puissant medium / high
Recherche web avec sources Support web search medium
