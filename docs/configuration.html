Configuration (panneau Neurow)

Tout se règle dans le panneau (pas dans les formules) : fournisseur par défaut, modèles, clés API, max tokens, et effort de raisonnement OpenAI.

Fournisseur par défaut

Le fournisseur par défaut s’applique à toutes les fonctions AI.*. Il se choisit via les boutons Gemini / OpenAI dans la section Configuration.

  • Avantage : vous pouvez partager un fichier Excel sans “polluer” les formules avec des options.
  • Avantage : changement de fournisseur “global” en 1 clic.

Max output tokens (réglage unique)

Neurow utilise un max output tokens partagé pour les deux fournisseurs. C’est votre principal levier de contrôle de coûts et de longueur de réponses.

  • 256–2048 : la plupart des usages “Excel” (résumé, traduction, classification, nettoyage).
  • 4096+ : cas longs (TABLE volumineux, explications riches, prompts complexes).
Pourquoi un réglage unique ? Cela évite des divergences entre fournisseurs et maintient un comportement cohérent quand vous basculez Gemini ↔ OpenAI.

Réglages par fournisseur

Gemini

  • Modèle Gemini : sélection via liste déroulante.
  • Clé API Gemini : stockée localement (voir section Stockage).
  • Boutons : Enregistrer, Effacer, Tester.

OpenAI

  • Modèle OpenAI : sélection via liste déroulante.
  • Effort de raisonnement : slider (valeurs usuelles : low, medium, high).
  • Clé API OpenAI : stockée localement.
  • Note : Neurow n’envoie pas de paramètres de sampling (temperature, top_p).
Effort de raisonnement. La documentation OpenAI présente principalement low/medium/high. D’autres valeurs peuvent être refusées par certains modèles. Si vous rencontrez des erreurs, revenez sur medium.

Stockage local (où sont mes clés ?)

Selon votre environnement Office, Neurow utilise :

  • Stockage local Office (préféré) si disponible ;
  • sinon stockage local du navigateur ;
  • et un fallback mémoire en dernier recours (rare).

Utilisation : tokens, coûts, et diagnostics

Le panneau affiche :

  • Tokens total / entrée / sortie ;
  • Coût estimé (indicatif) par fournisseur et par modèle ;
  • Activité sur la dernière heure ;
  • Logs des appels (durée, cache, erreurs, etc.).

Cache intelligent

Le cache est local et opère à deux niveaux :

  • Cache en mémoire (dans le runtime Office) : très rapide.
  • Cache persistant (stockage local) : survit aux redémarrages.

Le panneau expose les stats (entrées, taille) et permet de vider le cache.
