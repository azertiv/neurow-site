Excel + IA, directement dans vos formules

Des custom functions AI natives dans Excel — avec votre clé API.

AI.Native ajoute un namespace AI.* à Excel (ASK, WEB, TRANSLATE, CLASSIFY, FORMULA, TABLE…). Vous configurez une fois le fournisseur (OpenAI ou Gemini) et le modèle dans le panneau, puis vous utilisez l’IA comme une fonction Excel — dans n’importe quel classeur. Les requêtes partent directement d’Excel vers l’API du fournisseur : aucun serveur intermédiaire.

Fonctions AI.*

Une bibliothèque de custom functions IA

Des fonctions prêtes à l’emploi pour automatiser l’analyse, la rédaction et la mise en forme directement dans Excel.

  • ASK, SUMMARIZE, TRANSLATE, CLASSIFY, EXTRACT…
  • Fonctionne sur cellules et plages (A1:B50).
  • Sorties adaptées à Excel : texte ou matrices “spill”.

Ops

Cache, logs et contrôle des coûts

Un panneau “Ops” pour suivre l’activité, estimer la consommation, et diagnostiquer rapidement un classeur.

  • Cache intelligent pour limiter les tokens.

Vie privée : pas d’intermédiaire

AI.Native n’ajoute aucun serveur intermédiaire : vos requêtes partent de l’add-in vers l’API (OpenAI/Gemini), avec votre clé API. Les clés, le cache et les logs sont stockés localement sur votre appareil — et vous pouvez les effacer à tout moment.

Clés API locales : pas d’envoi vers un serveur tiers.
Paramètres centralisés : fournisseur, modèle et tokens se règlent dans le panneau.
Coûts maîtrisés : max tokens partagé + cache + déduplication.

FAQ

Les questions qui reviennent avant installation.

Mes données passent-elles par vos serveurs ?

Non. L’add-in appelle directement l’API du fournisseur (OpenAI/Gemini) depuis votre environnement Office. Aucun serveur AI.Native n’est interposé. Le stockage de la clé, du cache et des logs est local.

Pourquoi le fournisseur n’est-il pas dans la formule ?

Par design : le fournisseur (et le modèle) se choisit dans le panneau. Cela évite des fichiers illisibles, réduit les erreurs, et garde des formules stables et partageables.

Comment maîtriser les coûts ?

Trois leviers : (1) le réglage “Max output tokens”, (2) le cache local, (3) la déduplication et une limite de concurrence pour éviter les “bursts”.

Quels modèles sont supportés ?

AI.Native se configure via une liste de modèles dans le panneau. Vous pouvez choisir un modèle rapide/économique, ou un modèle plus puissant. Voir le wiki « Modèles ».
